# Tower Defense - 프로젝트 규칙

## 파일 구조
- `index.html` - 페이지 구조 + 스타일
- `game.js` - 게임 로직 전체 (단일 파일)
- `GUIDE.md` - 코드 구조 & 개념 가이드 (한글)

## 개발 규칙

### ⚠️ GUIDE.md 즉시 동기화 (항상 실행)
**`game.js`에 한 줄이라도 시스템/동작 변경이 발생하면, 동일한 응답 안에서 `GUIDE.md`를 반드시 갱신한다. 사용자가 별도로 요청하지 않아도 자동 실행이 기본값이다.**
- 예외 없음. 코드 변경 후 가이드 동기화를 누락한 채로 응답을 끝내지 않는다.
- 단순 오타/포매팅 수정처럼 **시스템 동작 영향이 0인 변경**만 동기화 생략 가능.
- 코드 수정과 GUIDE 갱신은 같은 turn에 묶어 처리 (다음 응답으로 미루지 않음).

### 업그레이드 시 필수 작업
1. `game.js` 수정 (기능 구현)
2. `GUIDE.md` 업데이트 (변경된 시스템 설명 반영) — 위 즉시 동기화 규칙 적용
3. `CLAUDE.md` 하단 업그레이드 로그에 기록 추가 (의미 있는 시스템/밸런스/UI 변경 한정)

### GUIDE.md 업데이트 원칙
- 새 시스템 추가 시 → 해당 섹션 새로 작성
- 기존 시스템 변경 시 → 해당 섹션 내용 수정
- 숫자(줄 번호, 수치 등)가 바뀌면 반드시 갱신
- 업그레이드 아이디어 체크리스트에서 완료된 항목 체크

### 코드 컨벤션
- Vanilla JS + Canvas API only (프레임워크 금지)
- 외부 라이브러리 사용 금지
- 한글 UI 텍스트 유지
- 모바일 + 데스크톱 입력 모두 지원

### 기술 스택
- HTML5 Canvas
- Vanilla JavaScript (ES6+)
- localStorage (최고 점수 저장)
